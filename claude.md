
### 🎯 核心目标：打造一个私有化的终极 JavDB 镜像
你需要一个能够全自动扫描本地硬盘、穿透反爬防御抓取精确元数据、利用本地 AI 进行影视级本地化翻译、并最终通过精美 Web 界面展示的媒体管理中心。

---

### 🧱 核心模块与需求清单

#### 1. 🕷️ 自动化智能刮削引擎 (Auto-Scraper)
* **无头自动化**：放弃脆弱的 HTTP 请求，全面采用 Playwright 驱动真实浏览器环境。
* **防盾穿透**：无缝绕过 Cloudflare 5 秒盾，并具备自动点击“18岁年龄确认墙”的极速非阻塞逻辑。
* **深度马甲探测**：在抓取详情页时，自动开新标签页潜入演员主页，扒取并下载高清**头像**，同时提取极其关键的**历史别名 (Aliases)**。
* **防封锁策略**：内置任务队列与随机休眠机制，伪装人类行为。

#### 2. 🧠 AI 本地化与数据富化引擎 (Local LLM NLP)
* **算力分离**：支持跨局域网调用独立的 Ollama 推理服务器（如 Sakura/Qwen），彻底释放爬虫机的压力，并解决跨机 IPv4/IPv6 监听绑定的网络运维问题。
* **物理级实体剥离**：在翻译前，利用刮削到的“主名+别名”黑名单，对日文标题进行精准的正则与字符串替换（斩首行动），彻底杜绝大模型产生“定语从句灾难”。
* **动态 Prompt 约束**：针对不同模型下发特定指令，压制大模型的“废话综合征”，强制只输出翻译结果。
* **结构化数据推理**：利用大模型将零散的 `series`（系列）进行语义聚类，甚至能直接从标题中推理出缺失的系列名，并输出严格的 JSON 格式。

#### 3. 💾 数据访问与持久化层 (DAL & SQLite)
* **多表级联关系**：抛弃扁平数据，建立 `videos`、`actors`、`tags` 以及多对多中间桥接表的标准关系型数据库。
* **无损数据迁移**：能够使用 Python 脚本执行 `ALTER TABLE`，在不伤及历史数据的情况下动态增加字段（如头像和别名）。
* **状态机驱动**：基于 `PENDING`、`SUCCESS`、`FAILED` 驱动整个业务流转。
* **DAL 层解耦**：所有 SQL 语句被严格封装在 `db_manager.py` 中，业务代码（爬虫/翻译）绝不直接触碰数据库。

#### 4. 🛡️ 工业级容错与防御机制 (Resilience)
* 处理大模型长列表聚类时，采用**三重阈值漏斗**：
    1.  SQL 频率阈值过滤（剔除低频错别字）。
    2.  NLP 字符串相似度过滤（传统算法快筛）。
    3.  大模型极低 Temperature 聚类 + 强制设置 `Uncategorized` 逃生舱。
* 完善的超时重试机制与数据批处理（Chunking）切片逻辑。

#### 5. 🌐 Web 展现层愿景 (Web UI)
* 系统需要向上提供标准的 RESTful API。
* 前端实现类似 JavDB 的高级瀑布流卡片展示、全局搜索、以及带有详情页
* 前端实现**修改**meta data 的功能, 用户可以点击 edit button 进入修改模式
* 前端实现**本地文件一键播放**的单页应用
* 前端实现**copy url**的功能, 这个 http 的 url 可以粘贴到播放器中 streaming 播放
* 前端实现**删除**功能, 当用户选择删除时, 把对应的视频文件删除, 但保留数据库信息, 加上删除的 flag
* 前端实现**添加** 功能, 当某些文件无法通过刮削方式获得信息时, 用户可以手工添加

#### 6. 本地文件meta data 的收集与整理
* 刮削本地文件时, 记录文件本身的meta data, 例如 creation time, last modify time, size 等等
